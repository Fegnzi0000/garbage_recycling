# 垃圾回收助手知识库（RAG）

> 本文件用于 LangChain4j RAG 检索增强：只放「公共规则/流程/说明」，不要放用户地址、手机号等隐私数据。

## 1. 系统能力概览

- 本系统提供：微信登录（含测试模式）、垃圾分类查询、地址管理、个人/高校/企业下单、订单管理。
- 服务统一前缀：`/api`

## 2. 垃圾分类与下单提示

- 分类接口：
  - `GET /api/category/top` 获取一级分类
  - `GET /api/category/{parentId}/sub` 获取二级分类
  - `GET /api/category/{categoryId}` 获取分类详情

- 个人下单接口：`POST /api/order/personal`
  - 必填：`addressId`
  - 可选：`scheduledTime`（建议格式 `yyyy-MM-dd HH:mm:ss`）
  - 可选：`items`（数组，物品明细）

- 地址接口：
  - `POST /api/address/create` 新增地址
  - `GET /api/address/list` 地址列表
  - `POST /api/address/default/{id}` 设置默认地址

## 3. 回收助手交互规则（安全）

- 助手只能帮助用户生成“下单草稿”，不会直接创建真实订单。
- 用户需要进行二次确认：调用后端确认接口提交草稿，才会真正创建订单。
- 助手回答中应避免泄露隐私信息：如手机号、详细地址等。

## 4. 常见问答（示例）

- 问：如何创建个人回收订单？
  - 答：先创建/选择地址（addressId），再提交订单数据（可包含预约时间、物品明细）。

- 问：我不知道属于哪个分类怎么办？
  - 答：可以先查看一级分类与二级分类列表，或描述物品材质/用途，助手会根据知识库与分类信息进行建议。
